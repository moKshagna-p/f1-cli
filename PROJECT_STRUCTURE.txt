KEY FEATURES OF NEW STRUCTURE
==============================

✅ Clear Separation of Concerns
   - API layer handles all HTTP communication
   - State layer manages data and changes
   - UI layer handles rendering
   - Utils provide shared functionality

✅ Modular Design
   - Each file has single responsibility
   - Easy to understand and modify
   - Simple to test individual modules
   - Good for collaboration

✅ Scalable Architecture
   - Easy to add new features
   - Easy to add new API endpoints
   - Easy to add new UI components
   - Easy to modify state management

✅ Professional Organization
   - Clean file structure
   - Clear module dependencies
   - Comprehensive documentation
   - Production-ready code quality

After (Organized):

- Easy to locate code
- Simple to add features
- Self-documenting structure

HOW TO NAVIGATE
===============

To understand the flow:
1. Start at index.js (entry point)
2. Read src/index.js (main class)
3. Follow the poll() method
4. See how it calls:
   - api.fetchAllTelemetry()
   - stateManager.updateState()
   - renderer.render()

To modify a feature:
1. If it's API-related → edit src/api/openf1Client.js
2. If it's state-related → edit src/state/stateManager.js
3. If it's display-related → edit src/ui/renderer.js, formatters.js, or colors.js
4. If it's configuration → edit src/utils/constants.js

To add a new feature:
1. API endpoint? → Add to src/api/openf1Client.js
2. New state? → Update src/state/stateManager.js
3. New display? → Update src/ui/renderer.js and formatters.js
4. New configuration? → Add to src/utils/constants.js
